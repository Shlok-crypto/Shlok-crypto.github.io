From Zero to Deployed: My Journey as a Meta Production Engineering Fellow

By Shlok Tomar | August 9, 2025

Hello everyone! My name is Shlok Tomar, and I'm a final-year Computer Science student at Washington State University with a passion for machine learning and building scalable, reliable systems. This past summer, I had the incredible opportunity to combine these interests as a Production Engineering Fellow through the Meta and MLH partnership. It was a deep dive into what it takes to keep massive services running, and I want to share one of the most rewarding parts of my experience: building a production-ready portfolio website from scratch.

What's a Production Engineer, Anyway?

Before the fellowship, my idea of a software engineer was mostly focused on building new features. But who makes sure those features can be reliably delivered to billions of users without breaking? That’s where Production Engineers (PEs) come in.

At Meta, PEs are a unique hybrid of software and systems engineers. They are the guardians of reliability and scalability. They write code to automate operations, design resilient systems, and dive deep into complex problems on live production environments. It's a role that perfectly matched my desire to not just build things, but to build things that last. The MLH Fellowship was my launchpad into this fascinating world.

My Project: Building a Production-Ready Portfolio

The core project of my fellowship was intensely practical: I was tasked with creating and deploying a production-ready portfolio website, applying all the core skills I was learning throughout the program. This wasn't just about making a pretty website; it was about treating my personal project with the same rigor as a real-world service.

Here’s a look at my journey from a folder of code to a fully deployed application:

  • Containerizing with Docker: The first step was to containerize my website using Docker. This was a crucial lesson in consistency. By creating a Docker image, I packaged my application, its dependencies, and its configuration into a single, portable unit. This eliminated the classic "but it works on my machine!" problem and was my first major step toward a professional workflow.
  • Hosting on Digital Ocean: I was given a budget to spin up my own virtual private server (VPS) on Digital Ocean. This meant I was in the driver's seat, responsible for everything from setting up the operating system and configuring the network to securing the server. It was a fantastic way to learn the fundamentals of system administration and what it truly takes to manage live infrastructure.
  • Orchestration with Docker Compose and Nginx: To run my website in a production-like environment, I used Docker Compose to manage my application's containers. I also set up an Nginx container to act as a reverse proxy. This powerful setup handled incoming web traffic, routed it to my application container, and allowed me to implement SSL certificates for secure HTTPS connections.
  • Automating with a CI/CD Pipeline: The most exciting part for me was building a complete Continuous Integration and Continuous Deployment (CI/CD) pipeline using GitHub Actions. This is where the magic of automation really clicked. Now, every time I push a change to my website's code on GitHub, a series of automated jobs kicks off. The pipeline automatically builds a new Docker image, pushes it to a container registry, and deploys the new version to my Digital Ocean server without any manual intervention. Seeing my changes go live within minutes of a git push was incredibly empowering.

Key Takeaways

This project was more than just a technical exercise; it was a mindset shift.

  • Think in Systems: I learned to think about the entire lifecycle of an application, not just the code.
  • Automation is Everything: At scale, you can't do things manually. Automating my deployment pipeline saved me time and reduced the risk of errors.
  • Own Your Infrastructure: Being responsible for my own server taught me invaluable lessons about security, maintenance, and the fundamentals of how the internet works.

This fellowship has been an amazing journey. It solidified my passion for building robust systems and gave me a clear vision for the kind of engineer I want to become. If you're a student fascinated by what happens after the code is written, I highly recommend exploring the world of Production Engineering.

Thanks for reading!
